# WordPress Landing Page Reference

Patterns for creating a WordPress landing page when the user prefers WordPress over Lovable.

---

## When to use WordPress

- User already has WordPress hosting (e.g., cyon.ch, Hostpoint, Infomaniak)
- User wants CMS flexibility for blog/content marketing
- User needs multilingual support (WPML/Polylang)
- Landing page needs complex content that changes frequently

## When NOT to use WordPress

- Speed is critical and user has no WordPress experience → use Lovable
- Simple one-page landing → Lovable is faster
- User wants to avoid hosting management → Lovable

---

## Recommended stack

```
Theme:      Astra or GeneratePress (lightweight)
Builder:    Elementor Free or Gutenberg + Spectra
Forms:      WPForms Lite or Fluent Forms
SEO:        Yoast SEO or Rank Math
Caching:    LiteSpeed Cache or WP Super Cache
Security:   Wordfence or Sucuri
Analytics:  Google Site Kit or Plausible
```

## Key setup for app integration

### Link buttons to the app

All CTA buttons must point to the Railway-hosted app:

```
"Jetzt registrieren" → https://app.[domain.tld]/register
"Anmelden"           → https://app.[domain.tld]/login
"Als Kandidat starten" → https://app.[domain.tld]/register?role=kandidat
"Als Arbeitgeber"     → https://app.[domain.tld]/register?role=arbeitgeber
```

### CORS configuration (if needed)

If the WordPress site calls the app's API:
```php
// In app's server (Railway), not in WordPress
// Already handled by Express CORS middleware
```

### Shared domain setup

```
domain.tld        → WordPress (A record to hosting IP)
app.domain.tld    → Railway (CNAME to railway target)
```

Both can share the same SSL provider. WordPress hosting handles SSL for the root domain, Railway handles SSL for the subdomain.

## SEO setup in WordPress

### Yoast SEO configuration
1. Install and activate Yoast SEO
2. Run the setup wizard
3. Set organization info (name, logo, social profiles)
4. Configure title format: `%title% — [App Name] | %sep% %sitedesc%`
5. Create XML sitemap (auto-generated by Yoast)
6. Submit sitemap to Google Search Console

### Schema.org (via Yoast)
- Set "Organization" as the site type
- Add SoftwareApplication schema via custom Schema block
- Enable breadcrumbs

### Performance
- Install LiteSpeed Cache or WP Super Cache
- Optimize images with ShortPixel or Imagify
- Disable unused plugins
- Use CDN (Cloudflare free tier works well)

## Landing page structure (WordPress)

Create these pages:

| Page | Slug | Purpose |
|------|------|---------|
| Home | `/` | Hero + features + CTA |
| Features | `/features` | Detailed feature descriptions |
| Pricing | `/pricing` | Plans and pricing |
| About | `/ueber-uns` | Team, mission, trust signals |
| Blog | `/blog` | Content marketing, SEO articles |
| Contact | `/kontakt` | Contact form |
| Privacy | `/datenschutz` | Privacy policy (legal requirement) |
| Imprint | `/impressum` | Legal imprint (required in DACH region) |

### Legal pages (DACH region — mandatory)
- **Impressum:** Company name, address, registration number, contact
- **Datenschutz:** Data processing, cookies, third-party services, user rights
- These are legally required for any commercial website in Germany, Austria, Switzerland, and Liechtenstein
