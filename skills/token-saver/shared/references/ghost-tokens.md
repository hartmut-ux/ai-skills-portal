# Ghost-Token-Hygiene (Hebel 5)

Quelle: Token Optimizer (alexgreensh), Token Optimizer MCP (ooples).
Ziel: 10-30% versteckte Token-Reduktion durch saubere Session-Hygiene.

## Was sind Ghost Tokens?

Tokens, die Context fressen, ohne dass jemand sie aktiv nutzt:
- Mega-Pastes vom User, die nur teilweise relevant sind
- Re-Reads derselben Datei in einer Session
- Zurück-quotierte System-Reminders
- Ungenutzte Tool-Definitionen
- Veraltete TodoList-Einträge
- Whitespace-Pollution in Dokumenten

## Sieben Hygiene-Regeln

### 1. Keine Mega-Pastes ohne Zielfrage

Wenn der User 50k Tokens Code pastet:
- ❌ Sofort losarbeiten
- ✅ Fragen: "Welcher Teil — welche Funktion / welche Zeile?"

Spart oft 40k+ Tokens beim Zurück-Quoten.

### 2. System-Reminders nicht zurück-quoten

- ❌ "Ich sehe einen System-Reminder, der sagt: [3000 Tokens System-Reminder zurück]"
- ✅ "Verstanden, ich passe an."

### 3. Tool-Listen gezielt laden

Wenn ToolSearch nötig:
- ❌ Query: "all tools" → 100+ Schemas geladen
- ✅ Query: "select:Read,Edit,Grep" oder gezielter Keyword

### 4. Keine Re-Reads

Datei, die gerade editiert wurde, NICHT nochmal `Read`-en. Der Harness trackt File-State.

### 5. Frühe Konsolidierung bei Multi-File-Edits

- ❌ Edit-Read-Edit-Read-Edit (jeder Read fügt Context hinzu)
- ✅ Plan erstellen → Batch von Edits in einem Schwung

### 6. TodoList klein halten

- Max 7 Items aktiv
- Completed-Tasks regelmässig wegräumen (alle 10 Updates checken)
- Keine Meta-Tasks wie "Plan erstellen für Plan-Erstellung"

### 7. Whitespace/Boilerplate aus Docs

Wenn du selbst Files schreibst:
- Keine doppelten Leerzeilen
- Keine 80-Zeichen-Banner aus `=========`
- Keine ASCII-Art-Headers
- Keine "Generated by..." Footer

## Detection: Wann ist meine Session "schwer"?

Symptome:
- Antworten werden langsamer
- Tool-Calls geben unerwartete Truncation
- User sagt "vergiss alles vorherige" oder "neue Session"
- Du wiederholst dich aus älteren Turns

→ Vorschlag an User: "Diese Session ist gerade context-schwer. Soll ich die Kernfakten in eine `session-state.md` zusammenfassen und du startest eine neue Session?"

## Compression-Patterns aus Token Optimizer MCP

Diese sind external (MCP-side), aber das Prinzip lässt sich emulieren:

- **Aggressive Caching:** Bei wiederkehrenden Lookups (gleiche API, gleiche Query) Antwort cachen — Claude kann das via Datei: `/tmp/cache/<hash>.json`
- **Diff-Compression:** Bei iterativen Edits nur Diffs zeigen, nicht ganze Files
- **Schema-Stripping:** API-Responses durch `jq` filtern, bevor sie Context erreichen

## Faustregel

Bei jedem Tool-Call fragen: **Bringt das Output mehr Wert als Tokens?**

Wenn nein → Filter setzen, anders fragen, oder ganz weglassen.
